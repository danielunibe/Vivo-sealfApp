# REGLAS DE PARCHES
- Futuros patches no deben volver a mover Ajustes arriba sin autorización explícita.
- Ajustes se mantiene integrado en el dock (ítem 5).
- Nunca usar ni convertir las opciones del menú de Ajustes en "tabs textuales" excesivamente largas.
- Registrar venta se mantiene siempre en el centro (ítem 3).
- La sección de Calendario no se debe alterar de su estado funcional/visual a menos que se solicite específicamente. Las cajas deben seguir siendo cuadrados redondeados, no círculos.
- Abstenerse de rediseñar toda la app o proponer múltiples cambios innecesarios cuando se pida un "patch" puntual (ejemplo: cambiar cajas pesadas para formularios).
- Todo parche debe concentrarse **en una sola Fase** del PIPELINE.md. No avanzar de fase ni programar generalidades sin resolver el objetivo de la fase actual. Primero validar entorno y compilar, luego diseñar y funcionales.
